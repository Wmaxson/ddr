Taco Queast

by: Max

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

This game was originally going to be a Dance Dance Revolution type game with arrow keys. But, as the game evolved, I like the idea of monsters as the games enemy, and it became this.

2. What I Learned

One of the suggestions I got was to add a boss. The boss was more aesthetic, as it did not change gameplay a ton, I still likes how it turned out. Another suggestion I got, was to make the game faster. The projectiles where going too slow, and it made the game dull. So, I made the game faster, and more exiting, but still relatively easy.

3. Architecture of My Game

The code runs, by having all of small pieces of the project feeding into larger ones. For example: The Projectile class is run in the Enemy class, which is run in the, EnemyManager class, which is run in the panel class, which is run in the runner class.

4. Technical Implementation

I am very proud that the game runs, and doesn't freeze upon opening.

5. Most Challenging Part

The most challenging part of the project was my stupidity, and my two brain cells not cooperating.

6. Future Fixes and Updates

I would like to display the boss's health, and add a death animation.
